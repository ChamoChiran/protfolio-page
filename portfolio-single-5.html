World History Search Pipeline

Application Preview

Project Date 2025
Role NLP, RAG, Vector Search
Tools Python, ChromaDB, LangChain
GitHub View Repository

Objective

Transform raw educational content from world history textbooks into a queryable vector database for intelligent retrieval and AI-powered tutoring. This semantic search system enables context-aware retrieval of historical information for research and educational applications.

Key Features

  • PDF extraction with chapter/page metadata preservation
  • Intelligent text cleaning and normalization
  • Semantic chunking using transformer embeddings
  • Vector database storage in ChromaDB for fast similarity search
  • Context-aware semantic search with ranking

Pipeline Architecture

The project follows a modular data pipeline approach:

  1. PDF Extraction: Convert textbooks to structured text
  2. Text Cleaning: Remove noise and normalize formatting
  3. Semantic Chunking: Group sentences by meaning
  4. Vector Database: Store and index embeddings
  5. Semantic Search: Query and retrieve relevant passages

Technology Stack

  • Python 3.10: Core programming environment
  • sentence-transformers: all-MiniLM-L6-v2 for embeddings
  • ChromaDB: Vector database for storage and retrieval
  • NLTK: Natural language processing toolkit
  • pymupdf: PDF text extraction
  • LangChain: Framework for LLM integration
  • Gemini API: AI-powered responses
  • Jupyter Lab: Interactive development

Use Cases & Roadmap

Current Applications:

  • AI-powered history context retrieval
  • Semantic research tools for educational content

Future Enhancements:

Example Query

from src.retrieval.search import search_vector_db

results = search_vector_db(
    query="How did the Neolithic Revolution change human societies?",
    k=5
)

for result in results:
    print(f"Chapter: {result['chapter']}")
    print(f"Text: {result['text']}\n")
